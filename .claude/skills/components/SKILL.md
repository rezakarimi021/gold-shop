# Components Design System Skill

# Purpose

You are responsible for every UI component in this project.

Every component must belong to one unified design system.

No component should feel like it was designed separately.

Users should immediately recognize consistency throughout the website.

---

# Design Philosophy

Every component should be:

Elegant

Minimal

Premium

Reusable

Accessible

Responsive

Predictable

Production Ready

---

# Core Principle

Never redesign an existing component unless there is a strong usability reason.

Consistency is more valuable than novelty.

---

# Component Standards

Every component must have:

Default State

Hover State

Focus State

Active State

Disabled State

Loading State (when applicable)

Error State (when applicable)

Success State (when applicable)

Responsive Behavior

Dark Background Compatibility

---

# Button Standards

Buttons must have:

Consistent height

Consistent padding

Consistent radius

Clear hierarchy

Comfortable spacing

Icons aligned perfectly

Loading indicator

Disabled appearance

Buttons should never resize while loading.

---

Primary Button

Highest visual priority.

One primary button per section whenever possible.

---

Secondary Button

Outlined or neutral.

Never compete with the primary action.

---

Ghost Button

Used only for secondary actions.

Minimal styling.

---

Danger Button

Reserved only for destructive actions.

Never use red for normal actions.

---

# Input Fields

Every input must contain:

Visible label

Placeholder (optional)

Helper text

Validation state

Error message

Success state

Focus state

Never rely only on placeholders.

---

# Search Field

Clear icon.

Comfortable padding.

Instant focus feedback.

Optional clear button.

Supports keyboard navigation.

---

# Dropdown

Smooth opening.

Keyboard accessible.

Selected item clearly visible.

Never overcrowd options.

---

# Checkbox

Large click target.

Visible checked state.

Keyboard accessible.

---

# Radio Button

Clear selected state.

Enough spacing between options.

---

# Toggle Switch

Simple.

Smooth.

Immediate feedback.

No unnecessary animation.

---

# FAQ Item

# Cards

Radius

Shadow

Spacing

Typography

Internal padding

# Icons

One icon family only.

Consistent stroke width.

Consistent size.

Aligned perfectly.

---

# Spacing

Every component follows the global spacing scale.

Never invent spacing values.

---

# Borders

Minimal.

Soft.

Elegant.

Never heavy.

---

# Shadows

Soft.

Natural.

Used only when helpful.

Never dramatic.

---

# Accessibility

Every component must support:

Keyboard navigation

Focus visibility

Screen readers

Proper semantic HTML

Adequate touch targets

Accessible contrast

---

# Performance

Components must:

Avoid unnecessary re-rendering.

Support lazy loading where appropriate.

Remain lightweight.

Reusable over duplicated.

---

# Final Validation

Before introducing a new component ask:

Can an existing component solve this problem?

Is it visually consistent?

Does it match the luxury brand?

Is it accessible?

Is it responsive?

Is it reusable?

Can another developer understand it immediately?

If not, redesign it.

Goal:

Every component should look like it belongs to the same premium design system, regardless of where it appears in the project.
