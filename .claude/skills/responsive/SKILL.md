# Responsive Design Skill

# Purpose

You are responsible for ensuring every page, component and interaction delivers an exceptional experience across all screen sizes.

Responsive design is not about shrinking layouts.

It is about redesigning the experience for each device.

Every screen must feel intentionally designed.

---

# Philosophy

Desktop

Laptop

Tablet

Mobile

Large Displays

All deserve first-class experiences.

Never create a desktop page and simply stack everything for mobile.

---

# Mobile First Thinking

While development may begin on desktop, every component must be evaluated independently for mobile usability.

Ask:

Can this interaction be comfortably used with one thumb?

Can this information be scanned quickly?

Can this section be simplified?

---

# Breakpoints

Design should adapt smoothly.

Avoid creating layouts that only work at specific widths.

Think in flexible ranges rather than rigid pixels.

---

# Layout Rules

Desktop

Multi-column layouts are encouraged.

Tablet

Reduce complexity.

Mobile

Prefer a single-column layout.

Never force users to zoom.

Never allow horizontal scrolling.

---

# Containers

Content should always remain centered.

Maximum width should prevent lines from becoming difficult to read.

Padding must scale with screen size.

---

# Grid System

Desktop

2–4 columns when appropriate.

Tablet

2 columns.

Mobile

1 column.

Never force four-column grids onto tablets.

---

# Navigation

Desktop

Full navigation.

Laptop

Condensed if necessary.

Tablet

Compact navigation.

Mobile

Drawer or bottom navigation when appropriate.

Never overcrowd the navigation bar.

---

# Hero Sections

Desktop

Large visual impact.

Tablet

Reduce image dominance.

Mobile

Focus on message first.

Keep the CTA visible without excessive scrolling.

---

# Typography

Typography must scale proportionally.

Never allow large headlines to occupy the entire screen.

Body text must remain readable.

Avoid tiny captions.

---

# Images

Images should scale without losing quality.

Never stretch images.

Maintain aspect ratio.

Crop intentionally.

Product photography must remain clear.

---

# Buttons

Buttons should always remain comfortable to tap.

Minimum comfortable touch target.

Consistent spacing between buttons.

Avoid placing destructive actions close to primary actions.

---

# Forms

Inputs should span available width on mobile.

Labels remain visible.

Keyboard should never cover important actions.

Validation messages should remain readable.

---

# Tables

Avoid horizontal scrolling whenever possible.

Transform large tables into cards on smaller screens.

Prioritize readability over preserving layout.

---

# Cards

Cards should adapt naturally.

Reduce unnecessary metadata.

Keep important actions immediately visible.

Maintain generous spacing.

---

# Product Grid

Desktop

Multiple products per row.

Tablet

Balanced layout.

Mobile

Comfortable scrolling with clear product hierarchy.

Never sacrifice product image size.

---

# Modals

Desktop

Centered.

Tablet

Responsive width.

Mobile

Full-screen when appropriate.

Never create tiny unreadable dialogs.

---

# Spacing

Reduce spacing carefully.

Never eliminate whitespace completely.

Luxury design still requires breathing room on small devices.

---

# Sticky Elements

Use sticky CTAs only when beneficial.

Avoid blocking content.

Avoid multiple sticky elements.

---

# Gestures

Support natural touch interactions.

Avoid hover-dependent functionality.

Everything important must remain accessible without hover.

---

# Performance

Optimize images.

Lazy load below-the-fold sections.

Avoid unnecessary JavaScript.

Animations should remain smooth on low-end devices.

---

# Accessibility

Keyboard navigation.

Visible focus.

Screen reader compatibility.

Proper semantic structure.

Large tap targets.

Readable contrast.

---

# Testing Checklist

Every page must be verified on:

Large desktop

Laptop

Tablet landscape

Tablet portrait

Large phone

Small phone

---

# Common Problems To Avoid

Do not allow:

Horizontal scrolling

Tiny buttons

Tiny text

Broken grids

Overlapping elements

Cut-off content

Hidden CTAs

Overflowing cards

Crowded layouts

Unbalanced spacing

---

# Luxury Mobile Experience

Luxury on mobile means:

Fast loading

Elegant spacing

Simple navigation

Beautiful product imagery

Comfortable reading

Confident interactions

No visual clutter.

---

# Final Validation

Before completing any page ask:

Can everything be used with one hand?

Is every action easy to reach?

Does the layout feel intentional?

Is readability preserved?

Does the page still feel luxurious?

Would I enjoy using this page on my own phone?

If the answer is no, redesign it.

Responsive design is complete only when every device feels like the primary device.
