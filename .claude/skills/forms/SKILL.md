# Forms Skill

# Purpose

You are responsible for designing every form in the project.

Forms should feel effortless.

Users should never become confused, stressed or frustrated while entering information.

Every form should increase trust and reduce friction.

---

# Design Philosophy

A premium form feels:

Simple

Calm

Fast

Elegant

Predictable

Helpful

Accessible

Never overwhelming.

---

# Core Principles

Ask only for information that is necessary.

Every field must have a clear purpose.

If a field is not required—

Remove it.

---

# Form Structure

Preferred order:

Title

↓

Short description

↓

Input fields

↓

Helper information

↓

Primary action

↓

Secondary action

Never mix unrelated fields.

---

# Labels

Every field must have a visible label.

Never use placeholders as labels.

Labels remain visible at all times.

Use short and clear wording.

---

# Placeholders

Placeholders are examples.

Not instructions.

Not labels.

Example:

0912xxxxxxxx

rather than

Enter your phone number here...

---

# Input Design

Inputs should have:

Comfortable height

Consistent padding

Rounded corners

Clear border

Visible focus state

Readable typography

Consistent spacing

Never create different input styles on different pages.

---

# Focus State

Focus should be obvious.

Use:

Gold accent

Soft shadow

Border color

Never rely only on color.

---

# Required Fields

Mark required fields consistently.

Avoid excessive required fields.

Ask only what is necessary.

---

# Optional Fields

Clearly indicate optional fields.

Do not surprise users.

---

# Validation

Validate as early as possible.

Provide feedback immediately.

Avoid waiting until form submission.

---

# Error Messages

Every error should explain:

What happened

Why

How to fix it

Examples:

Phone number must contain 11 digits.

Password must contain at least 8 characters.

Avoid generic messages like:

Invalid input.

---

# Success Feedback

Show confirmation immediately.

Examples:

Phone number verified.

Address saved.

Order completed successfully.

Success should be calm.

Not celebratory.

---

# Helper Text

Use helper text only when useful.

Keep it short.

Explain expectations before users make mistakes.

---

# Field Order

Group related information together.

Example:

Name

↓

Phone

↓

Email

↓

Address

↓

Notes

Never mix unrelated information.

---

# Buttons

One primary button.

Optional secondary button.

Primary action should always be visually dominant.

---

# Button Text

Use action-oriented labels.

Examples:

Save Changes

Continue

Place Order

Sign In

Create Account

Avoid vague wording.

---

# Search Forms

Search input should:

Focus quickly

Support keyboard shortcuts

Display recent searches

Allow clearing

Remain responsive

---

# Login Forms

Only essential fields.

Email or Phone

Password

Forgot Password

Remember Me

Primary CTA

Alternative login methods if supported.

---

# Registration Forms

Keep short.

Split into steps if necessary.

Avoid intimidating users.

---

# Checkout Forms

Group into logical sections.

Customer

Shipping

Billing

Payment

Review

Each section should feel manageable.

---

# Address Forms

Use autocomplete when available.

Support apartment/unit.

Validate postal code.

Never force unusual formatting.

---

# Password Fields

Allow show/hide password.

Display password requirements.

Support password managers.

---

# Phone Numbers

Accept local formatting.

Provide country selector when needed.

Validate intelligently.

---

# Date Inputs

Use date pickers where appropriate.

Avoid requiring manual formatting.

---

# Dropdowns

Keep option lists manageable.

Support search for long lists.

Highlight selected option clearly.

---

# Checkbox & Radio

Large click area.

Clear selection state.

Accessible with keyboard.

---

# File Upload

Clearly show:

Supported formats

Maximum size

Upload progress

Success state

Error state

Preview when appropriate.

---

# Loading State

Disable submit button.

Show progress.

Prevent duplicate submissions.

Never leave users wondering.

---

# Accessibility

Every field must support:

Keyboard navigation

Screen readers

Visible focus

Proper labels

Error announcement

Semantic HTML

---

# Mobile Forms

Large touch targets.

Correct keyboard types.

Sticky submit button when useful.

Minimal scrolling.

Comfortable spacing.

---

# Performance

Forms should feel instant.

Optimize validation.

Avoid unnecessary re-renders.

Preserve entered data when possible.

---

# Things To Avoid

Never:

Hide labels

Reset forms after errors

Use tiny inputs

Show unclear errors

Force unnecessary information

Disable paste in password fields

Block browser autofill

Make users repeat information

---

# Final Validation

Before approving any form ask:

Can it be completed quickly?

Is every field necessary?

Are errors easy to understand?

Is the submit action obvious?

Does the form feel calm and premium?

Would a first-time visitor complete it without confusion?

If not, simplify it.

Goal:

Every form should feel effortless, trustworthy and worthy of a luxury e-commerce experience.
