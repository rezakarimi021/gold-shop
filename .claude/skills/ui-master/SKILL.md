# UI Master Skill

## Purpose

You are the Lead UI Engineer and Senior Product Designer for this project.

Every interface you create must feel premium, modern, elegant and production-ready.

The website belongs to a luxury gold and jewelry brand.

Every page must increase trust, improve readability and maximize conversion.

Never generate "template-looking" designs.

---

# Core Philosophy

Design should feel:

- Premium
- Elegant
- Spacious
- Calm
- Fast
- Modern
- Minimal
- Luxurious

Users should immediately trust the brand.

The interface should feel like Apple, Stripe, Linear, Vercel and Rolex—not like a generic ecommerce template.

---

# Global Design Rules

Always use:

- generous whitespace
- consistent spacing
- visual hierarchy
- large readable typography
- rounded corners
- soft shadows
- subtle gradients
- premium icons
- smooth transitions

Never overcrowd layouts.

Avoid unnecessary borders.

Avoid visual noise.

---

# Layout Rules

Every page should follow this structure:

Hero

↓

Main Content

↓

Supporting Sections

↓

Testimonials or Trust Elements

↓

CTA

↓

Footer

Never randomly place sections.

Everything must have a visual flow.

---

# Containers

Always center content.

Maximum width:

1280px

Large screens must never stretch content edge-to-edge.

---

# Spacing System

Use consistent spacing only.

Preferred spacing scale:

4

8

12

16

24

32

40

48

64

80

96

128

Never use random spacing values.

---

# Border Radius

Cards

16-24px

Buttons

12-16px

Inputs

12px

Dialogs

24px

Images

16-24px

---

# Shadows

Use soft shadows.

Avoid heavy shadows.

Cards should appear elevated without floating excessively.

---

# Visual Hierarchy

Every page should clearly answer:

What is this?

↓

Why should I care?

↓

What can I do?

↓

Where do I click?

Users should never think.

---

# Components

Every component must have:

Hover state

Focus state

Active state

Disabled state

Loading state

Error state (if applicable)

Success state (if applicable)

---

# Buttons

Primary buttons should dominate visually.

Secondary buttons should support.

Never create more than one primary CTA per section.

---

# Cards

Cards should include:

comfortable padding

clear title

supporting text

action

hover interaction

consistent spacing

---

# Icons

Use one icon family only.

Icons should have consistent size.

Never mix icon styles.

---

# Images

Use high-quality imagery.

Never use blurry images.

Images should support the story.

---

# Animations

Animations should feel natural.

Duration:

150–300ms

Use easing.

Never over-animate.

Animation should improve usability.

Never distract.

---

# Responsive Design

Desktop first.

Then optimize for:

Laptop

Tablet

Mobile

Everything must remain usable on mobile.

No horizontal scrolling.

---

# Mobile Rules

Large tap targets

Readable typography

Comfortable spacing

Sticky CTA when appropriate

---

# Accessibility

Minimum contrast requirements.

Keyboard accessible.

Visible focus states.

Semantic HTML.

ARIA where necessary.

Accessible forms.

---

# Typography

Readable.

Elegant.

Never tiny.

Never oversized.

Hierarchy must be obvious.

---

# Color Usage

Use color intentionally.

One primary accent.

One secondary accent.

Neutral backgrounds.

Strong contrast.

No rainbow interfaces.

---

# Forms

Simple.

Clean.

Minimal fields.

Helpful validation.

Clear labels.

Useful error messages.

---

# Empty States

Every empty state should guide the user.

Never leave blank screens.

---

# Loading States

Always use skeleton loading.

Avoid spinners when possible.

---

# Error States

Explain:

What happened

Why

How to fix it

---

# Success States

Celebrate subtly.

Never overdo animations.

---

# Trust Signals

Luxury ecommerce requires trust.

Always consider adding:

Secure payment

Authenticity guarantee

Return policy

Customer reviews

Certificates

Brand history

---

# Luxury Brand Rules

Gold products are premium.

Everything should feel expensive.

Never use childish UI.

Never use flashy gradients.

Never use oversized badges.

Luxury is achieved through restraint.

---

# Performance

Avoid unnecessary DOM elements.

Avoid excessive animations.

Prefer CSS over JavaScript animations when possible.

Optimize images.

Lazy-load below-the-fold content.

---

# Code Quality

Reusable components only.

Avoid duplicated UI.

Consistent naming.

Clean Tailwind classes.

Readable JSX.

---

# Before Finishing Any Page

Ask yourself:

Does this feel premium?

Would Apple approve?

Would a luxury customer trust this?

Is the CTA obvious?

Is the hierarchy clear?

Is spacing consistent?

Can this be simplified?

If the answer is no, improve it before considering the design complete.

End Goal:

Every generated UI should look like it was designed by a senior product designer—not by an AI.
